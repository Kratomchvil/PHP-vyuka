GET vs POST

Dvě základní metody HTTP požadavku

GET

Data jsou v URL

stranka.php?jmeno=Jan&vek=17

  • Viditelná v adresním řádku
  • Omezená délka (~2000 znaků)
  • Pro čtení dat (vyhledávání, filtrování)
  • Lze uložit do záložek

POST

Data jsou v těle požadavku

stranka.php (data nevidíte)

  • Neviditelná v URL
  • Bez limitu velikosti
  • Pro odesílání dat (formuláře, přihlášení)
  • Nelze uložit do záložek

Tip: v DevTools (F12 → Network) si můžete prohlédnout oba typy požadavků

PHP běží na serveru

PHP kód nikdy neběží v prohlížeči

Požadavek přijde
→
PHP se spustí od začátku
→
Vygeneruje HTML
→
Odešle odpověď
→
PHP skončí!

Klíčový fakt: Každý požadavek = PHP se spustí úplně znovu.

Proměnné z předchozího požadavku neexistují.

Proto potřebujeme speciální způsoby, jak data mezi požadavky přenést ($_POST, $_GET, $_SESSION).

Superglobální proměnné

Speciální pole, která PHP automaticky naplní daty z požadavku

$_GET      // data z URL parametrů (?klíč=hodnota)
$_POST     // data z formuláře odeslaného metodou POST
$_SERVER   // informace o požadavku (metoda, URL, ...)
$_SESSION  // data uložená na serveru mezi požadavky

Jsou to asociativní pole — klíče odpovídají atributu name v HTML

<!-- HTML -->
<input name="jmeno" value="Jan">

// PHP po odeslání formuláře:
echo $_POST['jmeno']; // "Jan"

HTML formulář

Základní struktura

<form method="POST" action="">

    <label for="jmeno">Jméno:</label>
    <input type="text" id="jmeno" name="jmeno">

    <button type="submit">Odeslat</button>

</form>
  • method — GET nebo POST
  • action="" — kam se data odešlou (prázdné = na sebe)
  • name — klíč v $_POST / $_GET

Typy vstupních polí

type="text" — obyčejný text
type="email" — email (kontrola formátu)
type="number" — číslo (min/max)
type="date" — výběr data
type="radio" — výběr jedné možnosti
type="checkbox" — zaškrtávátko (více možností)
<select> — rozbalovací seznam
<textarea> — víceřádkový text
// Radio - stejný name = jen jedna volba
<input type="radio" name="pohlavi" value="muz"> Muž
<input type="radio" name="pohlavi" value="zena"> Žena

// Checkbox - [] = pole hodnot
<input type="checkbox" name="jazyky[]" value="PHP"> PHP
<input type="checkbox" name="jazyky[]" value="JS"> JS

Zpracování formuláře v PHP

if ($_SERVER['REQUEST_METHOD'] === 'POST') {

    // 1. Čtení a čištění dat
    $jmeno = trim($_POST['jmeno'] ?? '');
    $email  = trim($_POST['email'] ?? '');

    // 2. Validace
    if ($jmeno === '') {
        $chyby[] = 'Jméno je povinné.';
    }

    // 3. Pokud OK → uložit / zobrazit
    if (empty($chyby)) {
        // uložit data...
    }
}
  • ?? '' — výchozí hodnota, pokud klíč neexistuje
  • trim() — odstraní mezery na začátku a konci

Bezpečnost: XSS útok

Co když uživatel do formuláře napíše tohle?

<script>alert('Hack!')</script>

Bez ochrany prohlížeč spustí ten JavaScript!

Špatně

echo $_POST['jmeno'];

Výstup se vloží přímo do HTML

Správně

echo htmlspecialchars($jmeno);

Převede < na &lt; atd.

Pravidlo: Vždy htmlspecialchars() při výpisu uživatelských dat

Bezpečnost: Validace na serveru

HTML validace (required, type="email") nestačí!

Uživatel může:

  • Otevřít DevTools a smazat atribut required
  • Poslat požadavek přes curl/Postman bez prohlížeče
  • Změnit hodnoty selectu/radia na cokoliv
// Whitelist validace - ověřit, že hodnota je z povolených
$povoleneTridy = ['1.A', '2.A', '3.A', '4.A'];

if (!in_array($trida, $povoleneTridy, true)) {
    $chyby[] = 'Neplatná třída.';
}

Pravidlo: HTML validace = pro pohodlí uživatele. PHP validace = pro bezpečnost.

Životní cyklus formuláře

1
Uživatel otevře stránku GET → PHP zobrazí prázdný formulář
2
Uživatel vyplní formulář a klikne Odeslat
3
Prohlížeč pošle data na server POST
4
PHP přečte $_POST, validuje data…
↳ Chyba → zobrazí formulář znovu s chybovou hláškou
↳ OK → uloží data, přesměruje (redirect) → GET
5
Prohlížeč následuje přesměrování → zobrazí výsledek

PRG Pattern

Post / Redirect / Get

Proč po úspěšném uložení přesměrováváme?

Bez PRG

Uživatel klikne Odeslat (POST)

Data se uloží, zobrazí se výsledek

Uživatel zmáčkne F5 (refresh)

→ Prohlížeč odešle POST znovu!

→ Data se uloží podruhé!

S PRG

Uživatel klikne Odeslat (POST)

Data se uloží, PHP přesměruje (302)

Prohlížeč udělá GET na novou URL

Uživatel zmáčkne F5 (refresh)

→ Prohlížeč zopakuje jen GET

// Po úspěšném uložení:
header('Location: ' . $_SERVER['REQUEST_URI']);
exit;

Shrnutí

  1. HTTP = požadavek ↔ odpověď; GET (čtení) vs POST (odesílání)
  2. PHP běží na serveru, každý požadavek = nový start
  3. Data z formuláře čteme z $_POST / $_GET
  4. Atribut name = klíč v poli
  5. Vždy validovat na serveru (HTML validace nestačí)
  6. htmlspecialchars() při každém výpisu
  7. PRG pattern = po uložení přesměrovat
